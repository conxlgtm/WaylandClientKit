# Support

WaylandClientKit is pre-foundation. The project is usable for experiments,
framework work, and protocol validation, but the public surface can still move.

## Questions

Use GitHub Discussions if they are enabled. Otherwise, open an issue and label
it as a question.

## Bugs

Use the bug report template. Include the exact command, compositor/session, Swift
version, and logs.

## Security

Do not file public security issues. Follow [SECURITY.md](SECURITY.md).

## Feature And Protocol Requests

Use the protocol support template for new protocol work. Use the feature request
template for higher-level API or behavior proposals.

## Compositor Evidence

Use the compositor evidence template when reporting advertised protocols,
runtime paths, fallbacks, or compositor-specific behavior.
